# Getting Started

shadcn Web Components are standards-based custom elements built with Lit. Setup
has two parts:

1. Load the global design tokens so component colors inherit valid CSS values.
2. Import component modules so their `shadcn-*` elements are registered.

Component layout and interaction styles live inside each element's shadow root.
The global token stylesheet supplies colors, typography, radii, shadows, and
other values shared across those shadow roots. The reset is recommended for a
consistent page baseline, but it is optional.

## npm and a bundler

Use this path with Vite, Angular CLI, or another tool that resolves npm package
imports.

Install the package:

```bash
npm install @billy_mcdowell/shadcn-web-components
```

If you use Chart, also install its peer dependency:

```bash
npm install chart.js
```

Load the default global styles in your document head. Version the URLs alongside
the installed package version, or host copies of these files with your app:

```html
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/styles/tokens.css"
>
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/styles/reset.css"
>
```

Register every component from your JavaScript or TypeScript entry point:

```ts
import '@billy_mcdowell/shadcn-web-components';
```

For a smaller application bundle, import only the component modules you use:

```ts
import '@billy_mcdowell/shadcn-web-components/button';
import '@billy_mcdowell/shadcn-web-components/card';
```

Then use the registered elements in your templates:

```html
<shadcn-button variant="default">Save changes</shadcn-button>
<shadcn-button variant="outline">Cancel</shadcn-button>
```

You do not need an import map in this setup. The bundler resolves `lit`,
`@floating-ui/dom`, `focus-trap`, and `@lit/context` from the package
dependencies automatically.

## Dependencies

Runtime JavaScript dependencies are resolved differently depending on how you
deliver the library.

### npm and a bundler

`npm install @billy_mcdowell/shadcn-web-components` installs `lit`,
`@floating-ui/dom`, `focus-trap`, and `@lit/context`. Your bundler resolves them
alongside component modules — no import map or extra install step is required for
typical apps.

Chart is optional: install `chart.js` when importing
`@billy_mcdowell/shadcn-web-components/chart`.

| Components / subpaths | Required bare imports |
|-----------------------|----------------------|
| Most primitives (`button`, `input`, `badge`, etc.) | `lit` |
| `dialog`, `sheet`, `drawer`, `alert-dialog` | `lit`, `focus-trap`, `@lit/context` |
| `popover` | `lit`, `@floating-ui/dom`, `@lit/context` |
| `chart` | `lit`, `chart.js` |
| Root `dist/index.js` (all components) | all of the above |

### CDN and plain HTML

Distributed modules keep bare imports for the dependencies above. Add matching
entries to an import map before any module scripts. See [HTML and CDN](./?path=/docs/getting-started--docs#cdn-and-plain-html)
for the full map used by the production gallery.

**Troubleshooting:** if the browser console shows
`Failed to resolve module specifier "@floating-ui/dom"`, you are missing an import
map entry (CDN) or have not installed the package with npm (bundler apps).

## CDN and plain HTML

For a page without a build step, use the exact published files shown below. The
import map is required because the distributed modules intentionally retain bare
imports for `lit`, `@floating-ui/dom`, `focus-trap`, `@lit/context`, and
optionally `chart.js`. It must appear before the module script.

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>shadcn Web Components</title>

    <link
      rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/styles/tokens.css"
    >
    <link
      rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/styles/reset.css"
    >

    <script type="importmap">
      {
        "imports": {
          "lit": "https://esm.sh/lit@3.2.1",
          "lit/decorators.js": "https://esm.sh/lit@3.2.1/decorators.js",
          "@floating-ui/dom": "https://esm.sh/@floating-ui/dom@1.7.4",
          "focus-trap": "https://esm.sh/focus-trap@7.8.0",
          "@lit/context": "https://esm.sh/@lit/context@1.1.6",
          "chart.js": "https://esm.sh/chart.js@4.5.1",
          "chart.js/": "https://esm.sh/chart.js@4.5.1/"
        }
      }
    </script>
    <script
      type="module"
      src="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/index.js"
    ></script>
  </head>
  <body>
    <shadcn-button variant="default">Save changes</shadcn-button>
  </body>
</html>
```

The root `dist/index.js` registers every component and needs every import map
entry above. To register only a button, keep the `lit` entries and replace the
module script with:

```html
<script
  type="module"
  src="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/components/button/button.js"
></script>
```

Use the direct `dist` files rather than a transformed `+esm` package endpoint.
The direct root module preserves the registration side effects for every custom
element. Pin the package and Lit versions so production deployments only change
when you deliberately upgrade them.

See the complete working setup in the [production component gallery](./examples/example.html).

## Global styles and themes

`tokens.css` defines complete CSS color values such as
`--background: oklch(0.145 0 0)`. Keep the `oklch(...)` wrapper when overriding a
color; a channel-only value such as `0.145 0 0` is not valid when a component uses
`background: var(--background)`.

Set `data-theme="light"` on the root element to use the bundled light palette:

```html
<html lang="en" data-theme="light">
```

Override tokens after the default stylesheet to create your own theme:

```css
:root {
  --primary: oklch(0.62 0.19 255);
  --primary-foreground: oklch(0.985 0 0);
  --radius: 0.75rem;
}
```

The JavaScript subpath `@billy_mcdowell/shadcn-web-components/styles` exports Lit
`CSSResult` helpers for component authors. It is not a browser stylesheet and
should not be used in a CSS `@import` or `<link>`.

## Events and properties

Listen for component events with normal DOM APIs:

```html
<shadcn-button id="save">Save changes</shadcn-button>
<script>
  document.querySelector('#save').addEventListener('button-click', (event) => {
    console.log('Button clicked', event.detail);
  });
</script>
```

Use HTML attributes for static string and boolean values. Set JavaScript
properties when passing non-string values or controlling a component directly.

## Angular with npm

Install the package, load the global styles from your app shell, and import all
components or selected subpaths once from `src/main.ts`:

```ts
import '@billy_mcdowell/shadcn-web-components/button';

import { bootstrapApplication } from '@angular/platform-browser';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent).catch((error) => console.error(error));
```

Add `CUSTOM_ELEMENTS_SCHEMA` to each standalone component that uses custom
elements:

```ts
import { Component, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

@Component({
  selector: 'app-root',
  standalone: true,
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
  template: `
    <shadcn-button
      variant="default"
      [disabled]="isSaving"
      (button-click)="save()"
    >
      {{ isSaving ? 'Saving...' : 'Save changes' }}
    </shadcn-button>
  `,
})
export class AppComponent {
  isSaving = false;

  save(): void {
    this.isSaving = true;
  }
}
```

For an NgModule application, add `CUSTOM_ELEMENTS_SCHEMA` to the module's
`schemas` array instead. Use attributes for static strings, property binding for
dynamic values, and event binding for custom events.

## Angular with a CDN

To keep the component library out of the Angular bundle, put the same stylesheet
links and import map from the plain HTML example in `src/index.html`, followed by
the direct component module:

```html
<script
  type="module"
  src="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@latest/dist/components/button/button.js"
></script>
```

The Angular template still needs `CUSTOM_ELEMENTS_SCHEMA`. Do not also import the
same component from npm, because loading both sources can register the same custom
element twice.

## Agent skill for consumer applications

The repository includes a reusable
[`use-shadcn-web-components` Agent Skill](https://github.com/billymcdowell/shadcn-web-components/tree/main/.agents/skills/use-shadcn-web-components)
for building with the published npm package or CDN files. It helps an agent choose
components, inspect the published Custom Elements Manifest, use the correct
attributes, properties, events, slots, and compound markup, and configure plain
HTML, Angular, React, Vue, Lit, Svelte, and other frameworks.

Run the [`skills.sh` CLI](https://skills.sh/docs/cli) from the root of the
application where you want to use the component library:

```bash
npx skills add https://github.com/billymcdowell/shadcn-web-components --skill use-shadcn-web-components
```

The installer discovers the skill in `.agents/skills`, asks which detected
agents should receive it, and installs it for the current project by default.
Select Cursor, Claude Code, Codex, or any combination of them.

To install non-interactively for all three agents:

```bash
npx skills add https://github.com/billymcdowell/shadcn-web-components \
  --skill use-shadcn-web-components \
  --agent cursor \
  --agent claude-code \
  --agent codex \
  --yes
```

Add `--global` to either command to make the skill available across all of your
projects. Run `npx skills update use-shadcn-web-components` later to update an
installed copy.

### Cursor

Choose `cursor` in the installer. Let Cursor select the skill from the request,
or type `/` in Agent chat and choose `use-shadcn-web-components`.

See the [Cursor Agent Skills documentation](https://cursor.com/docs/skills).

### Claude Code

Choose `claude-code` in the installer. Claude can select the skill automatically
from its description, or you can invoke it explicitly with:

```text
/use-shadcn-web-components
```

Restart Claude Code if a newly installed skill does not appear in the current
session.

See the [Claude Code Skills documentation](https://code.claude.com/docs/en/skills).

### Codex

Choose `codex` in the installer. Codex can select the skill automatically, or
you can invoke it explicitly in the CLI, IDE extension, or app:

```text
Use $use-shadcn-web-components to add a settings form with the published package.
```

Run `/skills` or type `$` in the CLI or IDE to confirm that Codex discovered it.
Codex normally detects skill changes automatically; restart it if the skill does
not appear.

See the [Codex Agent Skills documentation](https://developers.openai.com/codex/skills).
