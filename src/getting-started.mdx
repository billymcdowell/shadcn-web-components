# Getting Started

shadcn Web Components are standards-based custom elements built with Lit. Setup
has two parts:

1. Load the global design tokens so component colors inherit valid CSS values.
2. Import component modules so their `shadcn-*` elements are registered.

Component layout and interaction styles live inside each element's shadow root.
The global token stylesheet supplies colors, typography, radii, shadows, and
other values shared across those shadow roots. The reset is recommended for a
consistent page baseline, but it is optional.

## npm and a bundler

Use this path with Vite, Angular CLI, or another tool that resolves npm package
imports.

Install the package:

```bash
npm install @billy_mcdowell/shadcn-web-components
```

Load the default global styles in your document head. Version the URLs alongside
the installed package version, or host copies of these files with your app:

```html
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/src/styles/tokens.css"
>
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/src/styles/reset.css"
>
```

Register every component from your JavaScript or TypeScript entry point:

```ts
import '@billy_mcdowell/shadcn-web-components';
```

For a smaller application bundle, import only the component modules you use:

```ts
import '@billy_mcdowell/shadcn-web-components/button';
import '@billy_mcdowell/shadcn-web-components/card';
```

Then use the registered elements in your templates:

```html
<shadcn-button variant="default">Save changes</shadcn-button>
<shadcn-button variant="outline">Cancel</shadcn-button>
```

You do not need an import map in this setup. The bundler resolves `lit` and the
package's other JavaScript dependencies.

## CDN and plain HTML

For a page without a build step, use the exact published files shown below. The
import map is required because the distributed modules intentionally retain bare
imports for `lit` and `lit/decorators.js`. It must appear before the module script.

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>shadcn Web Components</title>

    <link
      rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/src/styles/tokens.css"
    >
    <link
      rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/src/styles/reset.css"
    >

    <script type="importmap">
      {
        "imports": {
          "lit": "https://esm.sh/lit@3.2.1",
          "lit/decorators.js": "https://esm.sh/lit@3.2.1/decorators.js"
        }
      }
    </script>
    <script
      type="module"
      src="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/dist/index.js"
    ></script>
  </head>
  <body>
    <shadcn-button variant="default">Save changes</shadcn-button>
  </body>
</html>
```

The root `dist/index.js` registers every component. To register only a button,
keep the same styles and import map but replace the module script with:

```html
<script
  type="module"
  src="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/dist/components/button/button.js"
></script>
```

Use the direct `dist` files rather than a transformed `+esm` package endpoint.
The direct root module preserves the registration side effects for every custom
element. Pin the package and Lit versions so production deployments only change
when you deliberately upgrade them.

See the complete working setup in the [production component gallery](./examples/example.html).

## Global styles and themes

`tokens.css` defines complete CSS color values such as
`--background: oklch(0.145 0 0)`. Keep the `oklch(...)` wrapper when overriding a
color; a channel-only value such as `0.145 0 0` is not valid when a component uses
`background: var(--background)`.

Set `data-theme="light"` on the root element to use the bundled light palette:

```html
<html lang="en" data-theme="light">
```

Override tokens after the default stylesheet to create your own theme:

```css
:root {
  --primary: oklch(0.62 0.19 255);
  --primary-foreground: oklch(0.985 0 0);
  --radius: 0.75rem;
}
```

The JavaScript subpath `@billy_mcdowell/shadcn-web-components/styles` exports Lit
`CSSResult` helpers for component authors. It is not a browser stylesheet and
should not be used in a CSS `@import` or `<link>`.

## Events and properties

Listen for component events with normal DOM APIs:

```html
<shadcn-button id="save">Save changes</shadcn-button>
<script>
  document.querySelector('#save').addEventListener('button-click', (event) => {
    console.log('Button clicked', event.detail);
  });
</script>
```

Use HTML attributes for static string and boolean values. Set JavaScript
properties when passing non-string values or controlling a component directly.

## Angular with npm

Install the package, load the global styles from your app shell, and import all
components or selected subpaths once from `src/main.ts`:

```ts
import '@billy_mcdowell/shadcn-web-components/button';

import { bootstrapApplication } from '@angular/platform-browser';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent).catch((error) => console.error(error));
```

Add `CUSTOM_ELEMENTS_SCHEMA` to each standalone component that uses custom
elements:

```ts
import { Component, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

@Component({
  selector: 'app-root',
  standalone: true,
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
  template: `
    <shadcn-button
      variant="default"
      [disabled]="isSaving"
      (button-click)="save()"
    >
      {{ isSaving ? 'Saving...' : 'Save changes' }}
    </shadcn-button>
  `,
})
export class AppComponent {
  isSaving = false;

  save(): void {
    this.isSaving = true;
  }
}
```

For an NgModule application, add `CUSTOM_ELEMENTS_SCHEMA` to the module's
`schemas` array instead. Use attributes for static strings, property binding for
dynamic values, and event binding for custom events.

## Angular with a CDN

To keep the component library out of the Angular bundle, put the same stylesheet
links and import map from the plain HTML example in `src/index.html`, followed by
the direct component module:

```html
<script
  type="module"
  src="https://cdn.jsdelivr.net/npm/@billy_mcdowell/shadcn-web-components@0.1.0/dist/components/button/button.js"
></script>
```

The Angular template still needs `CUSTOM_ELEMENTS_SCHEMA`. Do not also import the
same component from npm, because loading both sources can register the same custom
element twice.
